# Phase 05.1 Plan 01: Inbound Twilio Migration — env, HMAC, webhook handler, queue type Summary

**One-liner:** Swap Meta Cloud API inbound path for Twilio form-encoded webhook with HMAC-SHA256 base64 signature algorithm.

## What Was Built

Replaced the entire Meta WhatsApp Cloud API inbound stack with Twilio's webhook format:

- **`src/env.ts`**: Swapped 4 Meta env vars (`WHATSAPP_PHONE_NUMBER_ID`, `WHATSAPP_ACCESS_TOKEN`, `WHATSAPP_APP_SECRET`, `WHATSAPP_VERIFY_TOKEN`) for 3 Twilio vars (`TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_WHATSAPP_NUMBER`). Count drops from 16 to 15.

- **`src/lib/hmac.ts`**: Replaced `verifyWhatsAppHmac` (hex HMAC on raw body with `sha256=` prefix) with `verifyTwilioSignature` (HMAC-SHA256 base64 over URL + sorted URL-decoded params). Constant-time comparison retained.

- **`src/routes/webhook.ts`**: Complete rewrite — removed GET hub-verification handler, added Twilio form-encoded body parsing via `URLSearchParams`, strips `whatsapp:` prefix from `From` field, uses `MessageSid` as dedup key, uses `TWILIO_WHATSAPP_NUMBER` directly for `to_phone`.

- **`src/queue/heartbeat.ts`**: Renamed `HeartbeatJobData.waMessageId` to `messageSid` with updated JSDoc (Twilio MessageSid). Dedup key now uses `data.messageSid`.
