## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between data tokens |
| sm | 8px | Row internal padding, compact table cell padding |
| md | 16px | Default element spacing, form field gap, card padding |
| lg | 24px | Section padding, nav item height padding |
| xl | 32px | Page horizontal padding, panel gap |
| 2xl | 48px | Major section breaks between page regions |
| 3xl | 64px | Page-level top/bottom margins |

Exceptions:
- Audio waveform bars: 2px gap between each of the 24 bars (not on the 8-pt scale — bars are a data visualisation unit, not a layout unit)
- Touch targets on toggle controls: minimum 44px height (accessibility — visually impaired user may use the dashboard on a tablet handed to the caregiver)

## Typography

Two font families, four roles. All sizes in px for CSS custom property declaration.

| Role | Family | Size | Weight | Line Height | Usage |
|------|--------|------|--------|-------------|-------|
| Body | IBM Plex Sans | 14px | 400 (Regular) | 1.5 | Setup form labels, Log page prose, error messages |
| Data | IBM Plex Mono | 14px | 400 (Regular) | 1.4 | Phone numbers, timestamps, heartbeat decision values, memory snippets |
| Label | IBM Plex Sans | 12px | 600 (SemiBold) | 1.3 | Nav items, page section headings, table column headers |
| Heading | IBM Plex Mono | 20px | 600 (SemiBold) | 1.2 | Page titles, agent state display, waveform caption |

Rules:
- IBM Plex Mono used for all values that originate from data: phone numbers, decision type, timestamps, cron expressions, session phase strings
- IBM Plex Sans used for all instructional copy, form labels, status messages, empty state text
- No third weight is permitted (never 700/Bold — terminal aesthetic uses weight contrast only between 400 and 600)
- Font size never below 12px — accessibility floor

Source: REQUIREMENTS.md FE-08, ROADMAP.md Phase 5 Plan 3.

## Color

Palette is strictly three roles. No additional brand colours.

| Role | Hex | Usage |
|------|-----|-------|
| Dominant (60%) | `#0D0D0D` | Page background, panel background, table row background |
| Secondary (30%) | `#1A1A1A` | Card surfaces, nav sidebar, modal backdrops, input field backgrounds, table alternate row |
| Accent (10%) | `#00FF88` | See reserved-for list below |
| Destructive | `#FF4444` | Delete contact confirmation button, remove routine action only |

Accent `#00FF88` reserved for:
1. Active waveform bars (when session state = `listening` or `playing`)
2. `interrupt` heartbeat decision badge/indicator
3. Primary CTA button background (Login "Connect", Setup "Save", Dashboard "Send Command")
4. Nav active-page left border indicator (2px left border on active route)
5. Focus ring on all interactive elements (2px outline, 2px offset)
6. Priority contact toggle — "on" state indicator dot

Secondary semantic colours for heartbeat decision coding:
- `interrupt` → `#00FF88` (accent — source: REQUIREMENTS.md FE-04, ROADMAP.md Phase 5)
- `batch` → `#FFAA00` (amber — source: REQUIREMENTS.md FE-04)
- `skip` → `#FF4444` (red — same hex as destructive — source: REQUIREMENTS.md FE-04)
- `silent` → `#555555` (muted grey — not in requirements but needed for completeness)

Text colours:
- Primary text: `#E0E0E0` (not pure white — reduces harsh contrast on pure black)
- Secondary text / metadata: `#666666`
- Placeholder text: `#444444`

Border colour: `#2A2A2A` (single border value used everywhere — never multiple border shades)

Source: REQUIREMENTS.md FE-04, FE-08, ROADMAP.md Phase 5 Plan 3.

## Component Inventory

### Waveform SVG (FE-03)

- 24 vertical bars rendered as `<rect>` elements inside a fixed `<svg width="120" height="32">`
- Bar width: 3px; gap between bars: 2px; total width = 24×3 + 23×2 = 118px (fits in 120px viewBox)
- Inactive state: all bars at 4px height, colour `#2A2A2A`, centred vertically
- Active state (session = `listening` or `playing`): bars animate height with staggered CSS keyframe animation; colour `#00FF88`; heights vary pseudo-randomly between 4px and 28px using animation-delay offsets per bar
- Animation: `keyframes waveform-pulse` — from `scaleY(0.15)` to `scaleY(1)`, duration 600ms, ease-in-out, alternate, infinite; each bar gets `animation-delay: calc({i} * 40ms)` where `{i}` is 0–23
- Session state `idle`, `composing`, `awaiting_approval`: inactive (grey, static)
- Session state `listening`, `playing`: active (green, animated)
- SSE `agent-state` event drives the active/inactive switch via zustand `sessionPhase` field

### Heartbeat Row (FE-04)

Each row in the live feed shows: `[decision badge] [timestamp] [from_phone] [body_preview]`

- Decision badge: 8px×8px filled circle + decision string label, colour from decision colour map above
- Timestamp: IBM Plex Mono 12px `#666666`
- From phone: IBM Plex Mono 14px `#E0E0E0`
- Body preview: IBM Plex Sans 14px `#666666`, truncated at 48 characters with ellipsis
- Row height: 40px; horizontal padding: 16px
- Feed capped at 100 rows (zustand slice logic: `[event, ...log].slice(0, 100)`)
- New rows slide in from top with CSS `transition: transform 150ms ease-out`

### Nav Sidebar

- Fixed left sidebar, width 160px
- Background: `#1A1A1A`
- Nav items: IBM Plex Sans 12px SemiBold, uppercase, letter-spacing 0.08em
- Active item: `#00FF88` 2px left border, text `#00FF88`
- Inactive item: text `#666666`, no border
- Item height: 44px (touch target)
- No icons — text labels only

### Form Fields (Login, Setup)

- Input background: `#1A1A1A`
- Input border: 1px solid `#2A2A2A`
- Input border on focus: 1px solid `#00FF88`
- Input focus ring: 2px outline `#00FF88`, 2px offset
- Input text: IBM Plex Mono 14px `#E0E0E0`
- Placeholder: IBM Plex Mono 14px `#444444`
- Label: IBM Plex Sans 12px SemiBold `#666666`, uppercase
- Input height: 44px; horizontal padding: 12px
- Form field gap (label to input): 8px
- Form row gap (field to field): 16px

### CTA Button

- Background: `#00FF88`
- Text: IBM Plex Sans 14px SemiBold `#0D0D0D` (black on green — high contrast)
- Height: 44px; horizontal padding: 16px
- Border radius: 2px (minimal — keeps terminal aesthetic)
- Hover: background `#00CC6A` (10% darker)
- Active/pressed: background `#00AA55`
- Disabled: background `#1A1A1A`, text `#444444`, border 1px solid `#2A2A2A`

### Destructive Button

- Background: transparent
- Text: IBM Plex Sans 14px `#FF4444`
- Border: 1px solid `#FF4444`
- Height: 44px; horizontal padding: 16px
- Border radius: 2px
- Hover: background `rgba(255, 68, 68, 0.1)`
- Requires confirmation modal before executing delete (see Copywriting Contract)

### Toggle (Priority Contact, Morning Briefing)

- Width: 40px; height: 24px; border radius: 12px
- Off state: background `#2A2A2A`, knob `#666666`
- On state: background `#00FF88`, knob `#0D0D0D`
- Transition: 200ms ease
- Touch target wrapper: 44px height minimum

### Data Table (Contacts, Log, Routines)

- Background: `#0D0D0D`
- Header row: background `#1A1A1A`, text IBM Plex Sans 12px SemiBold uppercase `#666666`
- Body row: alternating `#0D0D0D` / `#111111`
- Row height: 48px; horizontal padding: 16px
- Cell text: IBM Plex Mono 14px `#E0E0E0` for data fields, IBM Plex Sans 14px for prose fields
- Row hover: background `#1A1A1A`

## Page-Level Layout Contracts

### Login Page (FE-01)

Layout: centred card, no sidebar.
- Card width: 360px; background `#1A1A1A`; padding: 32px; border radius: 4px; border: 1px solid `#2A2A2A`
- Page title: IBM Plex Mono 20px SemiBold `#00FF88` — "VoiceApp"
- Subtitle: IBM Plex Sans 14px `#666666` — "Caregiver Dashboard"
- Field: phone number, type=tel, placeholder "+27 83 100 0000"
- CTA: "Connect" — full width within card
- On submit: store userId in zustand + localStorage key `voiceapp_user_id`; redirect to `/dashboard`

### Setup Page (FE-02)

Layout: single-column form with sidebar nav.
- Fields: language select (English / Afrikaans), location text input (for EskomSePush area), quiet hours from/to time inputs, morning briefing toggle
- POST target: `/api/settings` with Bearer token header
- CTA: "Save Settings"
- Cron expression for quiet hours rendered as IBM Plex Mono data text below the time inputs (read-only display, not editable directly)

### Dashboard Page (FE-03)

Layout: two-column split. Left column (60%): agent state panel. Right column (40%): voice command simulator.

Agent state panel:
- Session phase displayed as IBM Plex Mono 20px SemiBold `#00FF88` when `listening`/`playing`, `#666666` otherwise
- Waveform SVG centred below phase label
- Last spoken text: IBM Plex Sans 14px `#E0E0E0`, italic, max 2 lines, truncated with ellipsis

Voice command simulator:
- Label: "Simulate Voice Command" — IBM Plex Sans 12px SemiBold uppercase `#666666`
- Text input: full width, IBM Plex Mono 14px
- CTA: "Send Command" — POST to `/api/voice/command` with `{ userId, transcript, sessionId }`
- Response display: IBM Plex Mono 14px `#00FF88` for spoken response field, monospace JSON view for full response

### Heartbeat Feed Page (FE-04)

Layout: full-width table with fixed header, scrollable body. No sidebar interaction needed.
- Real-time via SSE from `/api/sse/heartbeat?token={token}`
- Decision colour coding: interrupt `#00FF88`, batch `#FFAA00`, skip `#FF4444`, silent `#555555`
- Columns: Decision, Time, From, Preview
- Empty state: "No heartbeat events yet. Waiting for WhatsApp messages..."

### Contacts Page (FE-05)

Layout: data table + add form below table.
- Table columns: Name, Phone (Mono), Priority toggle, Actions (Delete)
- Add form: two fields (Name, Phone) + "Add Contact" CTA
- Priority toggle: inline in table row — PATCH `/api/contacts/:id/priority`
- Delete: shows confirmation modal before DELETE `/api/contacts/:id`

### Routines Page (FE-06)

Layout: data table with human-readable labels.
- Table columns: Label, Cron (Mono), Type, Enabled toggle, Actions
- Cron expression rendered as IBM Plex Mono data text
- Human-readable label computed client-side: "0 7 * * 1-5" → "Weekdays at 7:00 AM"
- No inline edit for Phase 5 — read-only display with enable/disable toggle only
- Empty state: "No routines configured."

### Log Page (FE-07)

Layout: three stacked sections with section headings.
- Section 1: Message History — table with columns: Direction (in/out badge), From/To, Time, Preview
- Section 2: Heartbeat Audit — same data as FE-04 feed but static (pagination, not live stream)
- Section 3: Memory Schema Viewer — table with columns: Created, Summary preview, Embedding dimensions (always "1536")
- All timestamps in ISO 8601 format displayed as IBM Plex Mono 12px `#666666`

## SSE Integration Contract

| Endpoint | Event Type | Payload Fields | Consumer |
|----------|------------|----------------|----------|
| `/api/sse/agent-state?token={token}` | `agent-state` | `{ userId, phase, timestamp }` | Dashboard waveform + phase label |
| `/api/sse/heartbeat?token={token}` | `heartbeat` | `{ id, userId, decision, from_phone, body_preview, timestamp }` | Heartbeat feed rows |
| Both endpoints | `ping` | `"keep-alive"` | Ignored by client (keep-alive only) |

Bearer token passed as URL query parameter `?token=` — `EventSource` API does not support custom headers.

Source: RESEARCH.md Pattern 1 (Hono SSE Push), Pattern 2 (React SSE Subscription).
