# Plan 04-01: ElevenLabs TTS + WebSocket Audio Push

## Goal

Build the ElevenLabs TTS streaming wrapper and the per-user WebSocket connection registry so that spoken audio can be pushed to the correct device with proper framing.

## Why

Plans 04-02 and 04-03 both call `pushInterrupt(userId, text)` to deliver audio. This plan creates that function and the underlying `streamSpeech` machinery. Without it, the voice command route and cron briefing have nowhere to send audio.

## Scope

### Files to Create
- `src/tts/elevenlabs.ts` — ElevenLabsClient wrapper; exports `streamSpeech(text, userId)`
- `src/ws/connections.ts` — per-user WSContext Map; exports `registerConnection`, `getConnection`, `removeConnection`, `pushInterrupt`

### Files to Modify
- `src/env.ts` — add `ELEVENLABS_VOICE_ID_EN` and `ELEVENLABS_VOICE_ID_AF` to `REQUIRED_ENV_VARS` (ELEVENLABS_API_KEY is already present on line 8)
- `src/server.ts` — replace `wsConnections` import from `./ws/manager` with `registerConnection` / `removeConnection` from `./ws/connections` in the `/ws/session/:userId` WebSocket handler

### Files to Create (Tests)
- `tests/tts.test.ts` — 6 tests covering streamSpeech framing, model selection, and pushInterrupt delegation

## Tasks

### Task 1: Create src/ws/connections.ts

Create `src/ws/connections.ts` with the following exact exports:

```typescript
import type { WSContext } from 'hono/ws'

const connections = new Map<string, WSContext>()

export function registerConnection(userId: string, ws: WSContext): void
export function getConnection(userId: string): WSContext | undefined
export function removeConnection(userId: string): void
export async function pushInterrupt(userId: string, text: string): Promise<void>
```

Implementation rules:
1. `registerConnection` sets `connections.set(userId, ws)`.
2. `getConnection` returns `connections.get(userId)`.
3. `removeConnection` calls `connections.delete(userId)`.
4. `pushInterrupt` imports `streamSpeech` from `../tts/elevenlabs` and calls `streamSpeech(text, userId)`. If no connection exists for `userId`, log `[WS] pushInterrupt: no connection for ${userId}` and return — do NOT throw.

`pushInterrupt` is the sole public entry point for sending TTS audio from outside this module. No caller should ever call `ws.send()` directly.

### Task 2: Create src/tts/elevenlabs.ts

Create `src/tts/elevenlabs.ts`:

```typescript
import { ElevenLabsClient } from '@elevenlabs/elevenlabs-js'
// NOT: import { ElevenLabs } — that constructor throws TypeError at runtime
```

Lazy singleton pattern (same pattern as `src/tools/ambient.ts` — created on first use so tests can override env vars):

```typescript
let _client: ElevenLabsClient | null = null
function getClient(): ElevenLabsClient {
  if (!_client) _client = new ElevenLabsClient({ apiKey: process.env.ELEVENLABS_API_KEY! })
  return _client
}
```

Model and voice selection logic — implement as a pure helper used inside `streamSpeech`:

```typescript
function selectModel(language: string | null): string {
  return language === 'af' ? 'eleven_multilingual_v2' : 'eleven_flash_v2_5'
}

function selectVoiceId(language: string | null): string {
  return language === 'af'
    ? process.env.ELEVENLABS_VOICE_ID_AF!
    : process.env.ELEVENLABS_VOICE_ID_EN!
}
```

`streamSpeech` signature and behavior:

```typescript
export async function streamSpeech(text: string, userId: string): Promise<void>
```

Steps inside `streamSpeech`:
1. Fetch user language from Supabase: `supabase.from('user_profile').select('language').eq('user_id', userId).single()`. Use `null` if not found.
2. Resolve model and voice ID using `selectModel` and `selectVoiceId`.
3. Get `ws` from `getConnection(userId)` (imported from `../ws/connections`). If absent, log and return.
4. Send `ws.send(JSON.stringify({ type: 'audio_start' }))`.
5. Call `getClient().textToSpeech.stream(voiceId, { text, model_id: modelId, output_format: 'opus_48000_32' })`.
   - `output_format` is ALWAYS `'opus_48000_32'` — never `'mp3_44100_128'` or the SDK default.
6. Iterate the returned `ReadableStream<Uint8Array>`:
   ```typescript
   for await (const chunk of stream) {
     ws.send(chunk)  // chunk is already Uint8Array
   }
   ```
   If `ws.send(chunk)` throws (e.g., ArrayBuffer required), retry with `ws.send(chunk.buffer)`. Log which variant works and use it consistently.
7. Send `ws.send(JSON.stringify({ type: 'audio_end' }))`.
8. Wrap entire operation in try/catch — log errors, do NOT rethrow (audio failure must not crash the process).

Import `getConnection` from `../ws/connections` — this creates a mutual import between `tts/elevenlabs.ts` (uses `getConnection`) and `ws/connections.ts` (uses `streamSpeech` via `pushInterrupt`). Break the cycle using a dynamic import inside `pushInterrupt`:

```typescript
// In src/ws/connections.ts — pushInterrupt
export async function pushInterrupt(userId: string, text: string): Promise<void> {
  const { streamSpeech } = await import('../tts/elevenlabs')
  await streamSpeech(text, userId)
}
```

And `streamSpeech` imports `getConnection` statically from `../ws/connections` — the static import is safe because it is not the side that dynamically loads.

### Task 4: Write tests/tts.test.ts

Create `tests/tts.test.ts` using Bun `mock.module()` pattern. The file must mock the following before importing production code:

```typescript
// Mock ElevenLabsClient
const mockStream = mock(async function* () {
  yield new Uint8Array([1, 2, 3])
  yield new Uint8Array([4, 5, 6])
})
const mockTextToSpeechStream = mock(() => mockStream())
mock.module('@elevenlabs/elevenlabs-js', () => ({
  ElevenLabsClient: mock(function() {
    return { textToSpeech: { stream: mockTextToSpeechStream } }
  }),
}))

// Mock supabase
const mockLanguageSelect = mock(() =>
  Promise.resolve({ data: { language: 'en' }, error: null })
)
mock.module('../src/db/client', () => ({
  supabase: {
    from: () => ({
      select: () => ({
        eq: () => ({
          single: mockLanguageSelect,
        }),
      }),
    }),
  },
}))

// Mock ws/connections (capture sent frames)
const sentFrames: Array<string | Uint8Array> = []
const mockWs = { send: mock((data: string | Uint8Array) => { sentFrames.push(data) }) }
const mockGetConnection = mock(() => mockWs)
mock.module('../src/ws/connections', () => ({
  getConnection: mockGetConnection,
  registerConnection: mock(() => {}),
  removeConnection: mock(() => {}),
  pushInterrupt: mock(async () => {}),
}))
```

Six tests to write:

1. **audio_start frame before first chunk** — call `streamSpeech('hello', 'user-1')`, assert `sentFrames[0]` parses as `{ type: 'audio_start' }`.
2. **binary chunks sent** — assert `sentFrames[1]` is an instance of `Uint8Array` with content `[1, 2, 3]`.
3. **audio_end frame after last chunk** — assert the last item in `sentFrames` parses as `{ type: 'audio_end' }`.
4. **Afrikaans model selection** — override `mockLanguageSelect` to return `{ data: { language: 'af' } }`, call `streamSpeech`, assert `mockTextToSpeechStream` was called with the second argument containing `model_id: 'eleven_multilingual_v2'`.
5. **English model selection** — language `'en'`, assert `model_id: 'eleven_flash_v2_5'`.
6. **pushInterrupt delegation** — create a separate mock for `pushInterrupt` in `ws/connections.ts` and confirm calling `pushInterrupt('user-1', 'Test alert')` invokes `streamSpeech` with those arguments (test via the dynamic import chain or by spying on `streamSpeech` directly).
