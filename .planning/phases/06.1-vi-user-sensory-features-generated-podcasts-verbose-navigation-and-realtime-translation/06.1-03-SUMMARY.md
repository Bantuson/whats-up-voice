Verbose navigation tool using Google Maps Directions + Places APIs with Claude-generated environment-first waypoint descriptions delivered as WhatsApp voice messages; navigating SessionPhase with haversine-based 50-metre proximity detection, webhook location pin handling, and interruptible navigation with resume offer.

## What Was Built

### src/tools/navigation.ts (new)

Verbose navigation tool with four exports:

- `startNavigation(userId, destination, originLat?, originLng?)` — fetches walking route from Google Maps Directions API, enriches first 5 waypoints with nearby Places (80m radius), transitions session to `navigating`, calls `streamSpeech` with the first verbal environment description
- `updateLocation(userId, lat, lng)` — checks haversine distance to current waypoint end; if within 50m advances to next waypoint and delivers description; otherwise re-reads current waypoint description
- `stopNavigation(userId)` — clears `navigationSession`, transitions to `idle`
- `describeWaypoint(instruction, nearbyPlaces, distanceMetres)` — calls Claude with VI-specific system prompt (environment-first, no markdown, nearby places by name, one paragraph max 3 sentences)

Haversine function (`haversineMetres`) exported for testability.

### src/session/machine.ts (extended)

Added `navigating` SessionPhase to union (plus `translating` from 06.1-02 prerequisite since that parallel plan hasn't applied to this worktree).

New exports:
- `setNavigationSession(userId, nav)` — stores navigation session data in state
- `clearNavigationSession(userId)` — removes navigation session without clearing other state
- `setTranslationTarget` / `clearTranslationTarget` — translation helpers also added

TRANSITIONS extended: `idle`/`listening`/`playing` can enter `navigating`; `navigating` can enter `idle`, `navigating`, `listening`.

### src/routes/webhook.ts (extended)

Added Twilio location pin detection: reads `Latitude` and `Longitude` form fields, and if the user is currently in `navigating` phase, fires `updateLocation` asynchronously (fire-and-forget to keep 200ms webhook response).

Dynamic imports used for `getPhase` and `updateLocation` to avoid circular dependency at module init time.

### src/agent/classifier.ts (extended)

Added two FastPathIntent values: `start_navigation`, `stop_navigation`.

Navigation patterns inserted before web_search to prevent "help me get to X" matching `web_search`:
- `/help me (get|go) to|navigate to|take me to|directions? to|how do i get to|find my way to/i`
- `/stop navigation|cancel navigation|end navigation|stop (guiding|directions)|i'?m here|i have arrived/i`

### src/agent/orchestrator.ts (extended)

Three new tools added to `ALL_TOOLS` (total 13 tools):
- `StartNavigation` — destination required, originLat/originLng optional
- `StopNavigation` — no required parameters
- `DescribeCurrentWaypoint` — re-narrates current waypoint for "where am I" queries

Corresponding cases added to `executeTool` switch.

### src/routes/api.ts (extended)

Three new handling blocks added before LLM path:
1. `stop_navigation` fast-path → `stopNavigation(userId)` + spoken confirmation
2. `start_navigation` fast-path → destination extraction via regex replace + `startNavigation`
3. Navigation interruption intercept — when `navState.phase === 'navigating'` and `intent === null`, answers via orchestrator then appends "Say continue navigation to resume your route" offer

`sanitiseForSpeech` and navigation tool imports added.
